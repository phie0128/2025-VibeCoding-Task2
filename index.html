關於我（About 區占位內容）

這裡目前是「關於我」區塊的暫時版面，用來放個人背景、工作經驗與自我介紹等文字。正式內容會在下一階段透過 Section SDD 的 Prompt 產生。

未來可以在這裡介紹你的職涯歷程、擅長領域、設計與開發風格，或是其他足以代表個人特色的段落。例如從設計到前端的轉職故事、你在專案中扮演的角色、合作流程與價值觀等。

本占位內容的目的，是先把 HTML 架構與 Class 命名固定下來，好讓之後的 Section SDD 可以直接生成一整個新的 <section>，再由開發者貼回來取代這一區。

之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 section 程式碼整段貼在 SECTION:ABOUT-START 與 SECTION:ABOUT-END 這兩個註解中間， 完全取代現在這一區。

給開發者的小提示：
未來如果要更新 About 區，只要：
  1. 用 Section SDD 的 Prompt 產生「完整 About 區 section」。
  2. 複製產出的整個 <section id="about" ...>...</section>。
  3. 把原本這兩個註解之間的內容整段刪除後貼上新的 section。
不需要改動其他區塊或 CSS Slot，即可完成整區替換。

技能（Skills 區占位內容）

Skills 區未來會用來呈現你擅長的程式語言、框架、工具與工作流程，例如 HTML/CSS、JavaScript、前端框架、設計工具或版本控制等。現在先以簡單卡片與清單作為占位排版。

範例分類 · 前端技術
這裡之後可以列出核心前端技術與熟悉程度，例如「語意化 HTML、RWD 版型切版、CSS 架構設計、元件化思維」等等。
範例分類 · 工作流程
也可以放上你在實務專案中使用的流程與工具，例如「設計稿還原、與設計師協作、Git 工作流程、跨團隊溝通」等項目說明。
範例分類 · 其他加值能力
若有 UI/UX、資料視覺化、簡報製作或內容企畫等能力，也可以在這裡以條列或短句方式呈現，補充你的專業輪廓。

之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 Skills 區 section 程式碼整段貼在 SECTION:SKILLS-START 與 SECTION:SKILLS-END 這兩個註解中間， 完全取代現在這一區，不需要保留任何原本的占位卡片。

作品集（Projects 區占位內容）

這裡會用來展示你的代表性專案，例如前端切版作品、互動體驗頁面、設計轉前端實作示例，或是與客戶／公司合作的產品頁面。現在先用卡片與圖片佔位，確保 HTML 結構與 class 命名穩定。

預計放上專案名稱、負責項目與使用技術簡述。

未來可以在這裡描述專案背景、解決的問題與你在其中的角色，例如負責 RWD 切版、互動細節實作、與設計／後端協作等內容。

範例專案卡片 B（占位）

聯絡我（Contact 區占位內容）

Contact 區將會提供訪客聯絡你的方式，例如 Email、電話、GitHub、LinkedIn 或其他社群連結，方便對方洽談合作或詢問作品相關問題。

未來正式版本可以在這裡寫上一段邀請式文案，例如開放哪些類型的合作（網站設計、前端切版、長期專案等），以及大致回覆時間或聯絡偏好（例如先寄信再約線上會議）。

之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 Contact 區 section 程式碼整段貼在 SECTION:CONTACT-START 與 SECTION:CONTACT-END 這兩個註解中間， 完全取代現在這一區，保留 id="contact" 與語意結構即可。
